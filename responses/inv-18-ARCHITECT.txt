### main.go

```
package main

import (
    "net/http"

    "github.com/cornjacket/user-service/internal/userservice/handlers"
    "github.com/cornjacket/user-service/internal/userservice/store"
)

func main() {
    st := store.NewStore()
    h  := handlers.New(st)
    mux := http.NewServeMux()
    h.Register(mux)
    http.ListenAndServe(":8080", mux)
}
```

### main_test.go

Package `main`. Uses `httptest.NewServer` so no live port is needed. Test sequence:

1. POST `/users` with `{"name":"Alice","email":"alice@example.com"}` → assert 201, capture `id`.
2. GET `/users/{id}` → assert 200, body matches name and email.
3. GET `/users/{unknown-id}` (e.g. a fresh UUID) → assert 404.
4. PUT `/users/{id}` with `{"name":"Bob","email":"bob@example.com"}` → assert 200, body has updated fields.
5. DELETE `/users/{id}` → assert 204.
6. DELETE `/users/{id}` again (now gone) → assert 404.

All JSON is decoded with `encoding/json`; response bodies are closed after each request.

### Dependencies

- `github.com/google/uuid` — already in `go.mod` (used by the store)
- Standard library only for main.go and test

### Constraints

- The Goal mentions `handlers.NewMux(store)` — that function does not exist. The actual handlers API is `handlers.New(s Store)` + `h.Register(mux)`. Design follows the real implementation.
- `store.NewStore()` is the correct constructor (not `store.New()`).
- The module path in `go.mod` is `github.com/cornjacket/user-service`; imports must use this path regardless of the physical directory location.

## Acceptance Criteria

1. `POST /users` with a valid JSON body `{"name":"Alice","email":"alice@example.com"}` returns HTTP 201 and a JSON body containing a non-empty `id` field.
2. `GET /users/{id}` using the id from criterion 1 returns HTTP 200 and a JSON body with `id`, `name` ("Alice"), and `email` ("alice@example.com") matching the created record.
3. `GET /users/{unknown-id}` (an id that was never created) returns HTTP 404.
4. `PUT /users/{id}` using the id from criterion 1 with body `{"name":"Bob","email":"bob@example.com"}` returns HTTP 200 and a JSON body with updated `name` and `email` fields.
5. `DELETE /users/{id}` using the id from criterion 1 returns HTTP 204.
6. `DELETE /users/{id}` using the same id as criterion 5 (now deleted) returns HTTP 404.
