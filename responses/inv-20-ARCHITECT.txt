## Design

**Language:** Go (same module as lifecycle; targets Go 1.21)

**Package path:** `internal/iam/authz`

**Files to produce:**
- `authz.go` — all types, stores, constructor, and handler methods
- `authz_test.go` — table-driven httptest-based tests for all five endpoints

### Data model

```go
type Role struct {
    ID          string   `json:"id"`
    Name        string   `json:"name"`
    Permissions []string `json:"permissions"`
}
```

### Store and handler struct

```go
type handler struct {
    mu        sync.RWMutex
    roleStore map[string]Role     // keyed by role ID
    userRoles map[string][]string // keyed by user ID → []roleID
}
```

A single `sync.RWMutex` guards both maps. Read-only operations (GET /roles, GET /users/{id}/roles, POST /authz/check) acquire `RLock`; write operations (POST /roles, POST /users/{id}/roles) acquire the full `Lock`.

### Handler() constructor

```go
func Handler() *http.ServeMux {
    h := &handler{
        roleStore: make(map[string]Role),
        userRoles: make(map[string][]string),
    }
    mux := http.NewServeMux()
    mux.HandleFunc("/roles", h.handleRoles)
    mux.HandleFunc("/users/", h.handleUsers)   // catches /users/{id}/roles
    mux.HandleFunc("/authz/check", h.handleAuthzCheck)
    return mux
}
```

`/users/` uses a trailing-slash prefix match so one handler covers both `POST /users/{id}/roles` and `GET /users/{id}/roles`. The handler splits `r.URL.Path` on `/` to extract the user ID and confirm the path suffix is `roles`.

### Endpoint logic

**POST /roles:** decode body; reject empty `name` with 400; generate uuid ID; store in roleStore; respond 201 with full Role JSON.

**GET /roles:** acquire RLock; iterate roleStore; collect into a `[]Role` slice (stable iteration order not required); respond 200 with JSON array (empty array `[]` if none).

**POST /users/{id}/roles:** decode body; reject empty `roleId` with 400; acquire Lock; look up roleId in roleStore — if missing respond 404; append roleId to userRoles[userID] only if not already present (idempotent); respond 201 with empty body.

**GET /users/{id}/roles:** acquire RLock; look up userRoles[userID] (empty slice if absent); for each roleID fetch from roleStore (skip if missing); collect into `[]Role`; respond 200 with JSON array.

**POST /authz/check:** decode body; acquire RLock; look up userRoles[userId]; for each roleID fetch from roleStore (skip missing); scan `permissions` slice for exact string match; return 200 `{"allowed":true}` on first match, `{"allowed":false}` if exhausted.

### Error responses

All error bodies: `{"error":"<message>"}` with appropriate status code. `POST /authz/check` never returns 404 — unknown users yield `{"allowed":false}`.

### Dependencies

- `github.com/google/uuid` — already in go.mod
- stdlib only otherwise (`encoding/json`, `net/http`, `strings`, `sync`)

## Acceptance Criteria

1. `POST /roles` with `{"name":"admin","permissions":["read","write"]}` returns 201 and a JSON body containing `"name":"admin"`, `"permissions":["read","write"]`, and a non-empty `"id"` string.
2. `POST /roles` with a missing `name` field (e.g. `{}`) returns 400.
3. `GET /roles` returns 200 with a JSON array; after creating two roles the array contains exactly two entries matching the created roles.
4. `GET /roles` before any role is created returns 200 with an empty JSON array `[]`.
5. `POST /users/u1/roles` with `{"roleId":"<id of existing role>"}` returns 201.
6. `POST /users/u1/roles` with `{"roleId":"nonexistent-id"}` returns 404.
7. `GET /users/u1/roles` after assigning one role to u1 returns 200 with a one-element array containing that role's full object.
8. `GET /users/u1/roles` for a user with no assigned roles returns 200 with an empty array `[]`.
9. `POST /authz/check` with `{"userId":"u1","permission":"read"}` returns 200 `{"allowed":true}` when u1 has been assigned a role whose permissions include `"read"`.
10. `POST /authz/check` with `{"userId":"u1","permission":"delete"}` returns 200 `{"allowed":false}` when no assigned role contains `"delete"`.
11. `POST /authz/check` with an unknown `userId` returns 200 `{"allowed":false}`.
12. Assigning the same role to a user twice via `POST /users/{id}/roles` results in the role appearing exactly once in `GET /users/{id}/roles`.
